# A1 Headless Dogfood — vinctor-codex-hook

- **Date:** 2026-06-10
- **Version under test:** 0.1.0-preview (parity build; `dist/src/cli.js`, prebuilt)
- **Mode:** Headless. No live authorization service. Auth env intentionally unset,
  so a successfully-classified call returns `deny:missing_auth_env` (= the SUCCESS
  signal "this call is inside the boundary") and an unclassifiable call **abstains**
  (empty stdout). `allow` / `deny:action_denied` are exercised separately by the
  service-backed E2E (`test-integration/local-service-e2e.test.ts`).
- **Probes:** 46 across Bash classifiers/defaults, apply_patch, file tools,
  WebFetch/WebSearch, MCP, and fail-closed paths, plus a 4-probe non-disclosure
  spot check. Driven through the real CLI via `/tmp/probe.mjs`.

## Summary

Every documented classifier, default, apply_patch path, file/web/MCP surface, and
fail-closed path fired exactly as designed, and the non-disclosure invariant held
on every sensitive probe (no URL token, password, host, file path, channel id, or
payload — and no `grt_`/`aak_` — appeared in any output). The known, documented
design limits showed up as expected (first-token Bash dispatch: `sudo cat .env`
abstains; ordinary file edits abstain). No correctness surprises.

Legend: `MAPPED` = `deny:missing_auth_env` (in boundary, would call the service);
`ABSTAIN` = empty stdout (deferred to Codex's native flow).

## Non-disclosure spot check — PASS

Ran with auth env set + a dead endpoint, grepped full stdout for each sensitive
substring:

- WebFetch `https://u:PASSWD123@h.example.com/p?token=SEKRET_TOK` → no
  `SEKRET_TOK` / `PASSWD123` / `h.example.com`.
- apply_patch `Update File: SEKRET_DIR/.env` → no `SEKRET_DIR`.
- Read `~/.ssh/id_rsa` (cwd `/home/SEKRET_USER`) → no `SEKRET_USER` / `id_rsa`.
- MCP slack `{channel_id:"CHAN_SEKRET", text:"PAYLOAD_SEKRET"}` → no `CHAN_SEKRET` /
  `PAYLOAD_SEKRET`.
- `grt_LEAK_SEKRET` / `aak_LEAK` never appeared in any output.

All reason strings are the fixed `DENY_TEMPLATES` from `src/output.ts`. Invariant
holds across every surface.

## Limits confirmed (documented, not bugs)

- **First-token Bash dispatch.** `sudo cat .env`, `env X=1 cat .env`, and command
  chaining defeat classification — documented in `docs/configuration.md`
  ("Bash matching is literal, not a shell"). Not a regression; an explicit non-goal.
- **The big unmeasured risk is upstream, not here.** This dogfood proves the hook
  *classifies* every surface correctly. It does **not** prove Codex actually
  *fires* `PreToolUse` for apply_patch / MCP / file / web tools — that is
  version-dependent (openai/codex#16732, #17794) and must be measured against a
  real Codex build. See ROADMAP "Verify and harden … hook firing."
