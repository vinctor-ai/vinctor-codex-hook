## Non-disclosure

Ratchet tests assert `grt_`, `evt_`, raw tool input (command / patch path / MCP
field), and mapped scope never appear in `permissionDecisionReason`, across the
full decision matrix (permit / service-deny / service-unavailable / missing-auth /
malformed) for Bash, apply_patch, and MCP — plus the abstain path which emits no
reason at all.

## Known limitations (carried to ROADMAP)

- Codex fires `PreToolUse` reliably for `Bash`; `apply_patch` and most MCP tools
  have been unreliable on some Codex builds. The hook classifies them correctly
  when invoked but cannot force Codex to fire.
- A multi-path `apply_patch` is enforced as a single most-destructive match (one
  `/v1/enforce` call per the v1 contract).
- Live `allow` / `action_denied` require a running Vinctor service + grant.
